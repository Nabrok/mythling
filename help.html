Mythling Help

Prerequisites

Prerequisites:

  • A MythTV backend running version 0.26, 0.27, or 0.27.1.
  • A mobile device running Android 4.0 or later.

Setup:

Launch Mythling and access the Settings action menu. Basic configuration can be accomplished through the first three screens:

1. Playback Settings

If you choose to play media on your Device, then no further configuration is required in Playback Settings.

For playback on a MythTV frontend, Mythling uses the Frontend Control Socket interface, so for that you'll need to specify the mythfrontend host name or IP address along with the control port.

2. Network Settings

Definitely start with an internal network. For that you'll need to specify your mythbackend machine's host or IP address.

When you decide to set up access from an external network, then here you can also enter the externally-available host information. There are more details about this on the Mythling GitHub Wiki site.

3. Connections

Mythling streams content using the MythTV Services API. The Connections screen is where you specify the port on your backend machine where these services can be accessed.

If you have Apache running on your backend machine, then you may optionally enter the host and port for that. With a standalone backend web server you're able to use the Mythling media services for retrieving media lists and program data. The main advantage of using Mythling media services is that they can be significantly faster. The biggest drawback of using Mythling media services is that they require you to have Apache with PHP installed and running.

More about Mythling media services.

Note: for streaming actual content, Mythling always uses the MythTV services API.

Lastly in Connections, using the MythWeb Access slider you can control whether MythWeb is available in the action menu.
